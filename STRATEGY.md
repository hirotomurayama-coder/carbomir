# Carbomir 戦略整理（プロダクト方針・上位レイヤー）

> **位置づけ**: CLAUDE.md「プロダクト方針（アライメント結果）」を補完する、価値・課金・進化の上位仮説。2026-05 時点の **作業仮説** であり、検証で更新する。設計・営業・課金で迷ったらまずここを見る。

## 1. 提供価値の主語：「情報」から「成果」へ

- これまでの整理（と CLAUDE.md）は **インプット中心**（＝良い情報を渡す）。だが人は知識ベースが欲しくて買うのではなく、**何かを終わらせる/通す/決める** ために買う。
- 営業の軸は「良い情報をあげます」ではなく **「あなたの〇〇を仕上げます」**。
- 買い手が払うのは情報そのものでなく、**①一次情報を追う時間** と **②間違える/見落とすリスク** の肩代わり。

| | インプット軸（旧） | アウトカム軸（あるべき） |
|---|---|---|
| 主語 | 情報・知識ベース | 成果物・達成 |
| 性質 | ビタミン | 鎮痛剤 |
| 効果測定 | PV・滞在・MAU | 成果物が出たか・対応完了・エクスポート・継続フォロー |

## 2. なぜ買うのか（防衛可能な核）

**前提**：2026 年は生情報だけでなく **汎用的な要約・解釈も AI でコモディティ化** しつつある。防衛線は上がった。AI/無料に侵食されにくい核は 4 つ：

1. **アカウンタビリティ（編集責任）** — 法人 CradleTo が名前を出し、出典・確信度を明示。開示・取締役会資料に **引用できる**。AI の推測は引用できない。＝最強の課金理由。
2. **継続メンテナンス** — サブスクの本質は「情報」でなく「鮮度を保ち続ける運用サービス」。
3. **能動的な“見逃さない”** — 受動検索（AI が得意）でなく、変化を向こうから知らせる。
4. **日本語 × 日本企業含意** — 一次資料は英語。日本文脈の編集解釈。

→ **「編集部の論点」がプロダクトの心臓**。網羅性でなく **判断可能性** で戦う。ここが凡庸なら課金理由が消える。

## 3. ツールのアウトカム（判断を持たずに価値を出す）

ツールは **「決定」そのものは持たない**（会社固有・判断責任・説得＝人/コンサルの領域）。ツールが持つのは：

> **「決定できる状態」と「決定が腐らない状態」。判断はコンサルが持ち、ツールはその“前後”＝準備と維持を持つ。**

1. **① 判断の手前（readiness）** — 論点と選択肢が出揃い、議論を始められる状態。コンサル工数も下げる。
2. **② 稟議の弾薬（ammunition）** — 稟議・取締役会資料の **根拠パート**（比較・業界水準・出典）。推奨は人。
3. **③ 判断のあと＝維持（durability）★サブスクの核** — 決めたことが今も有効かを監視し続ける状態。コンサルが構造的にやれない領域。

## 4. ツール × コンサル（CradleTo）の分界線

```
[理解][比較][論点を出す]      ← 判断の手前  ◎ツール
            ↓
[戦略を決める][稟議を通す]     ← 判断そのもの ◎コンサル＋人
            ↓
[監視][鮮度維持][報告][守る]   ← 判断のあと  ◎ツール
```

- ツールは判断を **囲む**。コンサルは中央の **スパイク**（episodic）。ツールは常時。
- **ビジネスモデル**：ツール＝集客＋見極め＋リテーナー／コンサル＝高単価スパイク。「相談する→ carboncredits.jp/contact」が **ハンドオフ点**。
- **分界線を正直に引くこと自体が価値**：「ここから先は会社固有なので人が要る」とツールが言う＝信頼になり、同時にコンサル誘導になる。→ **文脈化した相談 CTA**（汎用「相談する」より強い）。

## 5. ユースケース：カーボンクレジット調達（具体）

### 調達ジャーニー × 分界線

| 段階 | ツールが出す | コンサルが出す |
|---|---|---|
| 目的確定 | 主張ルール整理（SBT/VCMI/CORSIA） | 御社の目的の言語化 |
| 要件定義 | タイプ/レジストリ/品質/国内外を比較で構造化 | 予算・リスク許容度への翻訳 |
| 品質・適格性 | CCP適格・additionality/permanence・過去事件 | 個別PJのデューデリ |
| 市場・価格 | 価格水準（相場観）・OffsetsDB | 調達ルート・価格交渉 |
| **量・タイミング・判断** | （持たない） | **★ここが判断** |
| 稟議 | 根拠・比較図表・業界水準・出典 | 推奨ストーリー |
| 開示・主張 | VCMI/SBT の主張ルール | 御社開示文の作成 |
| **継続監視** | **★delisting/CCP/規制変更/リバーサルのアラート** | （やれない） |

### 3 アウトカムの調達版

- **① readiness**：除去 vs 回避（`credit-type-comparison`/`engineered-removal-comparison`）、NbS品質（`nbs-quality-comparison`）、レジストリ（`vcm-registry-comparison`/`vcm-standards`）、適格性（`credit-eligibility`）、国内外（`jcredit-jcm-verra`）、他社事例（`microsoft-cdr-portfolio`/`apple`/`cdr-buyer-coalition-comparison`）。
- **② ammunition**：比較表・業界水準ベンチ・品質リスクの出典・価格水準。すべて出典/確信度つきで引用可。
- **③ durability（判断を腐らせるイベント＝監視対象）**：

| イベント | 例 | 監視元 |
|---|---|---|
| 手法改訂・廃止 | Verra 旧REDD→VM0048 | 年表 + entity `redd-plus`/`verra-vcs` |
| 品質ラベル判定変更 | ICVCM が CCP不適格判定 | entity `icvcm-ccp` + 年表 |
| 評判リスク事件 | 報道で特定タイプ炎上 | 年表 `2023-01-guardian-verra-redd` |
| 国際クレジット受入ルール | シンガポール5%・Article6相当調整・CORSIA適格 | 年表 + entity `paris-article-6-2`/`corsia` |
| 主張ルール変更 | SBTオフセット制限・VCMI改訂 | entity `sbti`/`vcmi` |
| 永続性・リバーサル | 森林火災・バッファプール | entity `permanence` |

### 情緒的ジョブ（殺し文句）

> 「あなたが買ったクレジットが“無効”や“炎上”になったら、**見出しになる前にあなたが知る**。」

③維持は実質、グリーンウォッシュ報道に対する **保険**。

## 6. 将来像と市場成熟：段階モデル

- **市場タイミング**：実行（調達/オフセット/計画策定）に達した日本企業はまだ薄い。一方、開示義務・CBAM・サプライチェーン圧で「理解/計画」層は厚く、execution へ卒業していく。→ **land-and-expand**（情報で接点を作り、成熟とともに上の層を開く）。
- **3 層モデル**：

| 層 | 提供 | コホート | 位置 |
|---|---|---|---|
| **Understand（今）** | 体系的情報・規制監視・編集論点 | 理解/計画（厚い） | ◎稼働中 |
| **Decide（橋）** | 調達レディネス・適格性・稟議弾薬 | 移行層 | △一部 |
| **Manage（将来）** | 保有クレジットの監視・適格性維持・開示出力＝ポートフォリオ管理 | 実行（薄い→拡大） | ✕未着手 |

- **★ポートフォリオ管理はピボットでなく「レンズの追加」**：
  > Manage ＝「③維持の監視」を“市場全体”から“自社が保有するクレジット”に絞り込んだもの。データモデルも監視エンジンも同じ。「保有（holding）」オブジェクトを知識グラフの上に乗せるだけ。
- 含意：**今③維持（適格性・規制監視・知識グラフ）に投資するのが最高レバレッジ**。現需要を満たしつつ将来基盤になる二重取り。「網羅情報」への投資では橋は架からない。

## 7. データ戦略

- **データは買わない（AlliedOffsets 含む）。網羅データは作り込まない。**
- Manage 層が必要とするデータ＝①**顧客の保有**（顧客が持ち込む）②**自前の知識グラフ**（手法/レジストリ/適格性/規制ステータス＝堀）③**公開・オープン**（CarbonPlan OffsetsDB〔統合済〕・各レジストリ公開情報・J-クレジット/JCM 公的データ・規制一次情報）。
- 「腐っていないか」の判定に要るのは価格網羅でなく **適格性・規制・手法の状態** ＝ 編集で作る部分。paid aggregator 不要、競合依存も回避。
- **結果**：Carbomir は **データ網羅で戦う会社ではない**。exroad/AlliedOffsets とは土俵を分ける。**判断・適格性・編集** で戦う。

## 8. 価格の扱い：「データ」でなく「文脈」で持つ

| | 正確な実勢価格（live feed） | 価格水準・相場観（editorial） |
|---|---|---|
| 中身 | 取引/スポットのリアルタイム値 | レンジ・方向感・代表指標（出典・時点つき） |
| 体制 | データオペレーション（重い） | 編集（堀） |
| Carbomir | ✕ やらない | ◎ やる |
| 担い手 | ブローカー/市場/exroad/AlliedOffsets | Carbomir |

- 公開ソースで成立：World Bank *State and Trends of Carbon Pricing*、Nasdaq/Puro の CORC 価格指数、EUA/各国オークション、METI/J-クレジット平均、報道スポットレンジ。
- entity・比較行列に「**価格水準**」を出典・時点つきの editorial 属性として載せる（鮮度シグナルと同じ規律）。
- 正確な執行価格が要る瞬間＝**調達実行** は、ブローカー/市場/CradleTo へ **ハンドオフ**。
- **意識的に捨てるトレードオフ**：price precision / liquidity は取りに行かない。

## 9. 差別化ポジション（1行）

> **Carbomir は「どのクレジットを買うべきか」は決めない。判断に必要な論点・比較・適格性が出揃った状態を作り、買った後に規制変更で“使えなく/ジャンク認定”されたら誰より先に知らせる。判断・交渉・稟議の物語は CradleTo の相談で。**

- vs exroad / AlliedOffsets：**網羅でなく判断、データでなく編集・適格性** で戦う。

## 10. 現時点の最小の賭け（需要テスト）

- いきなり Manage を作らない（需要がない）。
- 代わりに **「ウォッチリスト」＝ポートフォリオの胚**：ユーザーが「自分に関係する規制/手法/レジストリ」をフォロー → 変化を通知。情報ツールのまま「自分ごと化」需要（≒将来のポートフォリオ需要）を低コストで検証。フォロー対象を将来「保有クレジット」に差し替えればそのまま Manage 層になる。
